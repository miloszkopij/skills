# Universal Contract Skills

Three portable, self-contained contract-review skills for AI coding agents. They work unchanged in **Claude Code**, **OpenAI Codex**, and **OpenCode** — plain markdown instructions plus one stdlib-only Python helper, no platform-specific tools, paths, or pip installs.

Specialized for a **Poland/EU-based buyer**: comments in Polish, GDPR/RODO, Polish Civil Code, UODO, Schrems II — with suggested replacement wording in the contract's original language, ready to paste into a counter-proposal.

## The skills

### `legal-risk` — general contract review

Substitute legal advisor for T&C, SaaS agreements, NDAs, employment contracts, vendor and licensing agreements, and open-source license reviews. Captures the buyer's actual use case (use type, data exposure, output destination, criticality) before analysis and re-grades every finding's severity contextually. Findings are tagged 🔴 deal-breaker / 🟠 negotiate / 🟢 acceptable / ⓘ informational.

### `dpa-analyzer` — deep DPA specialist

Goes far deeper than `legal-risk` on Data Processing Addendums: full GDPR Art. 28(3) compliance gap analysis, multi-entity controller structures, sub-processor chains, SCC/DPF/TIA transfer mechanisms, TOMs, audit rights, breach-notification timing benchmarks, DSR assistance, and data return/deletion. Prefer it over `legal-risk` for any DPA.

### `it-vendor-review` — IT operational risk

Surfaces the operational and commercial red flags a pure legal review misses: SLA math (uptime thresholds, credit formulas, real payout exposure), software-license audit traps and true-up mechanics, hardware EOL risk, pricing escalation, cloud egress/lock-in, data sovereignty, and vendor exit strategy. Prefer it over `legal-risk` for any IT/tech vendor contract.

## Output

Every skill produces a **markdown risk report** saved to disk: Polish executive summary (severity counts, top deal-breakers, sign/negotiate/walk-away recommendation, compliance checklist), then findings in document order, each with an exact section reference, verbatim anchor quote, Polish comment, and suggested replacement wording.

On request, each skill additionally produces an **annotated `.docx`** — the full original contract with real Word comments anchored to the relevant clauses and the executive summary prepended as page one. This is generated by the bundled `scripts/docx_tool.py`, which needs only Python 3 (standard library, no packages):

```bash
python3 scripts/docx_tool.py extract  contract.docx                      # read text out of a .docx
python3 scripts/docx_tool.py create   contract.md working.docx           # build a .docx from markdown
python3 scripts/docx_tool.py annotate working.docx findings.json out.docx  # inject Word comments + summary
```

## Repository structure

```
legal-risk/         SKILL.md + references/ + scripts/docx_tool.py
dpa-analyzer/       SKILL.md + references/ + scripts/docx_tool.py
it-vendor-review/   SKILL.md + references/ + scripts/docx_tool.py
```

Each folder is fully self-contained — copy just the one you need.

## Installation

### Claude Code

```bash
# user scope (all projects)
cp -r legal-risk dpa-analyzer it-vendor-review ~/.claude/skills/

# or project scope
cp -r legal-risk dpa-analyzer it-vendor-review <project>/.claude/skills/
```

Verify with `/skills`, or just ask: *"review this contract"*.

### OpenAI Codex (CLI with skills support)

```bash
# user scope
cp -r legal-risk dpa-analyzer it-vendor-review ~/.codex/skills/

# or project scope
cp -r legal-risk dpa-analyzer it-vendor-review <project>/.codex/skills/
```

### OpenCode

```bash
# project scope
cp -r legal-risk dpa-analyzer it-vendor-review <project>/.opencode/skill/

# or global
cp -r legal-risk dpa-analyzer it-vendor-review ~/.config/opencode/skill/
```

### Fallback for any agent (no skills support)

Reference a skill directly from `AGENTS.md` (or `CLAUDE.md`):

```
When asked to review a contract, read skills/legal-risk/SKILL.md and follow it exactly.
```

> Directory conventions occasionally change between tool releases — if a skill isn't picked up, check your tool's docs for the current skills path, or use the AGENTS.md fallback, which works everywhere.

## Usage

Trigger by name ("legal-risk", "dpa-analyzer") or naturally:

- *"Is this contract OK to sign?"* → `legal-risk` (or `it-vendor-review` for IT vendor paper)
- *"Is this DPA OK? What should we negotiate?"* → `dpa-analyzer`
- *"Check the SLA in this agreement"* → `it-vendor-review`

Input can be a `.docx`, a `.pdf`, pasted text, or a URL. The skill asks a short round of context questions (use case, data exposure, criticality) and re-grades severities accordingly before producing the report.

## Notes

- **Legal currency:** the skills verify cited authorities (DPF status, SCC 2021/914, GDPR/KC articles) via web search when the environment allows it. Without web access they proceed and explicitly mark citations as unverified.
- **Perspective:** always the buyer/controller signing the vendor's paper. Poland/EU is the home jurisdiction.
- **Scope disclaimer:** these skills are analysis aids, not a substitute for a qualified lawyer's sign-off on high-stakes agreements.
